Night of a show: GitHub kicks off show-night at 6:30pm ET → the job runs on this PC → polls Phish.net every ~75 seconds → each new song goes through the SQLite dedupe, gets formatted with its stats block, and posts to Truth Social — while site.py pushes the same setlist to mikeside.com. Jam milestones fire at 20/30/40 minutes; set recaps post between sets; the show recap + estimated lengths thread post when the window closes.
Next morning: verified-recap pulls exact track times from phish.in's recording and posts the verified lengths thread.
State: botstate.db rides in the Actions cache between runs, so restarts and swaps never double-post. Dashed = phase 2.
